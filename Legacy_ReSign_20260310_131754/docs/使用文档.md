# 使用文档（Pine + seccomp）

## 1. 你会得到什么

ReSign 是一个在 Android 设备上运行的重打包工具。它会对目标 APK 做注入与入口替换，并在目标进程启动时执行 payload：

- Java 层：使用 Pine 做“签名一致性”Hook
- Native 层：使用 seccomp + SIGSYS 做 IO/路径视图重定向（open/readlinkat 等）

---

## 2. 快速开始

### 2.1 构建并安装 ReSign

- 用 Android Studio 打开仓库
- 选择 `app` 模块
- 连接真机（建议 arm64）
- 运行 `app`（Debug 即可）

### 2.2 在 UI 中重打包目标应用

- 打开 ReSign
- 搜索/选择目标应用
- 按需选择功能开关
- 点击“重打包”

重打包输出位置：

- 主要输出：系统 Download 目录（或应用外部目录的 fallback）
- 同时会在应用私有目录保留一份 export 结构（便于调试）

---

## 3. 功能开关说明（运行期 meta-data）

重打包阶段会写入 Manifest meta-data，payload 运行期读取这些键来决定启用哪些能力：

- `resig.enableJavaHook`
   - true：启用 Pine 签名一致性 Hook
   - false：不做签名一致性修复

- `resig.enableNativeHook`
   - true：启用 Native 能力（包括 IO 重定向）
   - false：完全关闭 Native 能力

- `resig.enableIoRedirect`
   - true：启用 IO/SVC 重定向（需要 enableNativeHook=true）
   - false：关闭 IO 重定向

- `resig.enableMapsHide`
   - true：启用 maps 过滤（/proc/<pid>/maps，best-effort，需要 enableNativeHook=true 且 enableIoRedirect=true）
   - false：关闭 maps 过滤

- `resig.enableResourceRedirect`
   - true：启用资源重定向（实验能力）
   - false：关闭

- `resig.debugLog`
   - true：输出 payload 调试日志
   - false：尽量静默

说明：

- 当前 native 后端固定为 seccomp + SIGSYS，不再提供 Dobby/多后端选择。

---

## 4. payload dex（classesx.dex）如何生成与更新

### 4.1 什么时候需要更新

只要你修改了根目录的 `HookApplication.Java`（payload 入口类），都必须重新生成 `classesx.dex`，否则重打包注入的还是旧逻辑。

### 4.2 如何更新

在仓库根目录执行：

```bash
./build.sh
```

脚本会：

- 编译 `HookApplication.Java`
- 将 Pine 依赖打入 payload dex
- 输出 `out/dex/classes.dex`
- 同步到 `app/src/main/assets/classesx.dex`

重打包时会对 `classesx.dex` 做一次轻量“占位符写入”：

- 写入包名 / 伪签名（base64）/ 原 Application / 原 AppComponentFactory
- 写入逻辑在 `Inject.kt` 的 `editShellDEX(...)`，使用 dexlib2 重写 dex 字符串常量

## 5. 常见问题

### 5.1 重打包成功但目标应用启动崩溃

排查顺序建议：

- 打开 `resig.debugLog=true`，看 payload 是否执行、Pine hook 是否安装成功
- 目标应用是否多进程：如果签名检查发生在非主进程，需要确保 payload 在该进程也会执行
- 关闭资源重定向（如果开启了）：`resig.enableResourceRedirect=false`
- 如果日志显示 origin.apk 读取/校验异常：清理目标应用数据，或删除 `files/origin.apk` 后重试（payload 会自动重新解包）

### 5.2 Hook 没生效

- 确认 `resig.enableJavaHook=true`
- 确认你运行了 `./build.sh` 并重新重打包
- 检查 `HookApplication` 日志里是否有 Pine hook 安装成功的输出
